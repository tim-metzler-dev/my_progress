## Today I learned:
1. 	1. A key in web development is to make the website accessible for everyone. You can use the guidelines known as POUR:
		P- Perceivable: The user must be able to perceiv the information of your website. Like adding additional text to pictures for screen readers.
		O- Operable: The user must be able to interact with the UI. You can achieve that like making everything accesible with a mouse and or a keyboard for example.
		U- Understandable: It's helpfull to avoid complex sentences and use simple language as much as possible.
    R- Robust: A wide range of browsers and other tools must be able to interpret the content.
2. There are many types of different input devices like joy sticks, braille keyboards, touchpads and voice control for example. There are a ton of that. To ensure that your website,
   is truelly accessible to everyone, you can use tools such as Google Lighthouse (this is good because you can check also your software that isn't uploadet to the internet)
   or for more accuracity pagespeed.web.dev but that only can see uploadet websites with url. You can also use WAVE wich is an chrome extansion or  IBM Equal Accessibility Checker.
   You see there are much different opertunities to ensure your website is accessible for everyone. But keep in mind, that these are only tools and only have limited tests.
